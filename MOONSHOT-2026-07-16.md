# The next moonshot: prove the thesis with your own number

**Status:** designed, not built. Feasibility verified live 2026-07-15 (below).
**Why now:** every other number in this repo measures a *part*. This one measures
*the claim*.

---

## The hole

The thesis is one sentence:

> Memory has no intrinsic quality. It is only as good as the context it surfaces
> and the output it drives.

**Nobody has measured it.** Not us, not anyone.

| what we measure today | what it proves |
|---|---|
| `battery` — P@3 0.692, MRR 0.603 | retrieval ranks well *against its own labels* |
| `judge-bench` — Qwen 0.962 vs GPT-5 0.808 | Qwen is a good *judge* |
| `rot` — 12 classes | the store contains rot |
| decay AUC 0.781 | decay predicts *your kills* |
| run cards — 4/6 verified | agent claims survive reality, **n=6** |

Every one is a *component* metric. **None measures the delta Helicon causes.**
The closest thing in the README is `91.6% → 71% task completion, full-history vs
pruned (arXiv 2606.10209)` — **someone else's number, about someone else's
system, cited because we don't have our own.**

A judge can ask "so does it work?" and the honest answer today is "the parts
work."

## The experiment

Ask an agent a question whose answer **Oscar has personally ruled on**. Give it
memory retrieved two ways. Measure how often it gets the ruled answer right.

```
                  ┌─ A: RAW store    (7,283 memories, rot included)  ─┐
question ─────────┤                                                   ├─→ judge vs Oscar's ruling
(Oscar has ruled) └─ B: CLEANED store (4,107 live, rot retired)     ─┘
```

- **Ground truth = the human ruling.** Not synthetic, not an LLM's opinion, not a
  label we invented. Oscar sat down and decided `lea birthday = 07-18`.
- **The rot is real and still there.** The raw store still contains `07-13`.
- **The independent variable is exactly Helicon's work**: the 3,176 retirements.

If B beats A, that number is the product. If it doesn't, **that is worth knowing
more than anything else in the repo**, and we should find out before a judge or
an investor does.

## Build sketch (~1-2 sessions)

1. **A raw retrieval path.** `_load_all_embeddings` filters
   `review_status IN ('approved','pending')` — that filter *is* the treatment.
   Add a `include_retired=True` variant. **This is the only real code.**
2. **Probes from rulings.** Each ruling becomes a question + an answer key.
   `audit_log.human_decision = 'resolved:<truth>'`, and `details.dates` carries
   the rot. Generate: *"When is Lea's birthday?"* → key `07-18`, distractor
   `07-13`.
3. **Two contexts, one model.** Retrieve top-5 each way. Same model, same prompt,
   temperature 0. Qwen (it's a Qwen hackathon and `judge-bench` proved Qwen judges
   at frontier accuracy for 1/50th the cost).
4. **Score.** Did the answer match the ruling? Report A vs B, per probe.
5. **`helicon prove`** — one command, one number, reproducible.

## The honest risks, named up front

- **n=9 is small.** It is a *direction*, not a proof, and must be labelled that
  way — the same discipline that keeps "33% of agent claims are false" (n=6) off
  camera. It grows every time Oscar rules.
- **It might show no delta.** If retrieval already ranks the true memory above
  the rot, retiring the rot changes nothing — and *that finding is the most
  valuable one available*, because it would mean the retirement machinery earns
  its keep somewhere other than where we claim.
- **It could flatter itself.** The rulings were made *because* the exam surfaced
  those conflicts, so the probes are drawn from rot Helicon already found. That
  is a real selection effect. Name it; do not launder it.
- **A ruling is not a fact.** Oscar can be wrong. The ground truth is "what the
  operator decided", which is the honest thing to optimise for and should be said
  in those words.

## Why this and not the alternatives

- **macOS app** — builds and runs, and showcases neither Qwen nor Alibaba. Design
  says 10-12 sessions. It is the prettiest surface and the least on-brief.
- **The human bottleneck** (315 open vs 58 ruled, filling 5.4x faster than it
  drains) — the real product-market truth and the thing auto-triage exists for.
  But it is a *product* problem, and this is the *thesis*.
- **`review --terminals`** — Oscar's felt need and the sharpened north star
  (registry #017). Ship it. It is not a moonshot; it is the roadmap.

**This one is the moonshot because it is the only experiment whose result could
falsify the product.** That is what makes it worth running.
